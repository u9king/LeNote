#### 1.灰盒Greybox

> 别称：Blockout

- 白盒（Whitebox）：它是 Blockout 的初期阶段。设计师使用最简单的基础几何体（通常是白色的立方体、球体）来快速搭建空间骨架，重点在于验证空间尺度、玩家视线和基本动线。
- 灰盒（Greybox）：它是 Blockout 的进阶版本。在白盒的基础上，设计师会加入更复杂的几何体，并进行简单的灯光测试和初步的玩法功能验证（如掩体高度、跳跃距离）。





​	这是一个游戏开发术语，指在设计关卡时，使用简单的几何体（如立方体、圆柱体）快速搭建出场景的轮廓和结构，用来测试玩法、动线和尺度，之后再替换成精美的美术资源。



对于开发者（尤其是独立开发者）来说，直接做精美的模型是危险的。

1. **低成本修改：** 如果发现关卡太长或太短，修改一个方块只需要几秒钟，但修改一个高模建筑可能需要几天。
2. **专注玩法：** 避免被华丽的美术效果干扰，先确保关卡“好玩”。
3. **性能测试：** 可以在没有任何资源压力的情况下，测试关卡的视距和遮挡剔除（Occlusion Culling）。



**总结：** 它们都是在“没有美术资产（模型/贴图）”的情况下，用简单的形状搭建关卡的过程。



| **特性**     | **白盒 (Whitebox)**            | **灰盒 (Greybox)**                   |
| ------------ | ------------------------------ | ------------------------------------ |
| **主要目标** | 确定空间大小、比例、行走路径。 | 验证玩法细节、光影氛围、关卡节奏。   |
| **视觉呈现** | 只有最基础的方块（Cubes）。    | 包含楼梯、窗户形状、基础的光影。     |
| **功能性**   | 主要是“能不能走通”。           | 包含“掩体够不够高”、“跳跃能否够到”。 |
| **迭代速度** | 极快，随拆随建。               | 较快，开始进入微调阶段。             |
